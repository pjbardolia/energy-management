# Development Rules

1. Add comments in all code.

2. Maintain backward compatibility.

3. Never delete working code without approval.

4. Preserve existing API routes.

5. Use SQLAlchemy ORM.

6. Use PostgreSQL.

7. Use FastAPI.

8. Follow existing project structure.

9. Write production-ready code.

10. All endpoints must include validation.

11. Multi-company architecture must be preserved.

12. Document all major changes.


## Learning Mode

13. Explain all code changes.

14. Add comments for beginners.

15. Never assume user understands FastAPI or SQLAlchemy internals.

16. Provide testing steps after code changes.

17. Provide rollback steps before major changes.
